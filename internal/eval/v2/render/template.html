Methodology limits

  1. Accuracy is assigned by the configured LLM judge using the official GroupMemBench Correct/Incorrect prompt; judge model and thinking level are recorded in runtime provenance.
  2. Token F1, exact, and safe-success remain deterministic diagnostics.
  3. System cost scope is {{.CostScope}}; judge cost is recorded separately, and provider-internal extractor and embedding calls may be excluded.
  4. This report contains point estimates from this run. Repeated trials, judge calibration, and confidence intervals are required for stable claims.
